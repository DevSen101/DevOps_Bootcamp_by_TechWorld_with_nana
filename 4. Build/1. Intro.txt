## Build Process
Compiles source code + dependencies into single deployable **artifact** (e.g., JAR, WAR, ZIP). Reduces hundreds of files to one movable unit for production servers.[1][4]

## Artifact Repositories
Central storage for artifacts with versioning/backup:
- **Nexus** (Sonatype): Java-focused, free OSS version
- **JFrog Artifactory**: Universal repo (Docker, Maven, npm), enterprise-grade
- **Others**: AWS CodeArtifact, GitHub Packages[3]

New artifact per deploy ensures reproducibility.

## Language-Specific Examples
| Language | Build Tool | Package Manager | Artifact |
|----------|------------|-----------------|----------|
| Java     | Maven, Gradle | Maven Central | JAR/WAR/EAR |
| Node.js  | npm/Yarn   | npm registry   | tgz |
| Python   | pip/setuptools | PyPI       | wheel |
| .NET     | NuGet         | NuGet.org      | nupkg |[1][2]

**Key**: Build tools automate packaging; package managers handle deps.[4]
