## üéØ Docker Revolutionizes Artifacts
```
**TRADITIONAL**: JAR + WAR + ZIP + wheel ‚Üí Multiple repos/formats
**DOCKER**: **ONE IMAGE** = Code + Runtime + OS + Config
**SINGLE REPO**: Docker Hub / Harbor / ECR / GCR
```

## üöÄ Docker Build Pattern (All Languages)

```dockerfile
# JAVA ‚Üí Docker
FROM openjdk:17-jre-slim
WORKDIR /app
COPY target/app.jar app.jar
EXPOSE 8080
CMD ["java", "-jar", "app.jar"]

# BUILD: mvn package ‚Üí docker build -t myapp:1.0 .
```

```dockerfile
# NODE.js ‚Üí Docker  
FROM node:18-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --only=production     # Install deps IN image
COPY dist/ .                      # Pre-built JS
EXPOSE 3000
CMD ["npm", "start"]
```

```dockerfile
# PYTHON ‚Üí Docker
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install -r requirements.txt  # Install deps IN image
COPY . .
CMD ["python", "app.py"]
```

## üî• Complete DevOps Flow
```bash
# 1. BUILD APP
mvn package          # ‚Üí app.jar
npm run build        # ‚Üí dist/
pip install -e .     # ‚Üí ready

# 2. BUILD DOCKER IMAGE
docker build -t myapp:1.0 .
docker tag myapp:1.0 myregistry.com/myapp:1.0

# 3. PUSH TO REGISTRY (ONE REPO!)
docker push myregistry.com/myapp:1.0

# 4. DEPLOY (NO SERVER SETUP!)
docker run -d -p 8080:8080 myregistry.com/myapp:1.0
```

## ‚úÖ Docker vs Traditional Comparison

| Aspect | Traditional | Docker |
|--------|-------------|--------|
| **Artifact** | JAR/WAR/ZIP/wheel | **ONE IMAGE** |
| **Server Setup** | Install Java/Node/Python | **NONE** |
| **Dependency Mgmt** | npm install on server | **RUN in Dockerfile** |
| **Repo** | Nexus/JFrog per lang | **ONE Docker registry** |
| **Portability** | Platform issues | **Guaranteed** |
| **Rollback** | Keep old JARs | `docker run myapp:1.0` |

## üõ†Ô∏è Multi-Language Monolith Example
```dockerfile
# React + Node API
FROM node:18 AS frontend
COPY frontend/package*.json ./frontend/
RUN cd frontend && npm ci
COPY frontend/ ./frontend/
RUN cd frontend && npm run build

FROM node:18 AS backend
COPY backend/package*.json ./backend/
RUN cd backend && npm ci
COPY backend/ ./backend/
COPY --from=frontend /frontend/build ./backend/public/

EXPOSE 3000
CMD ["cd", "backend", "&&", "npm", "start"]
```

## üí° Docker Pro Tips
```
‚úÖ **Multi-stage builds** = Smaller images
‚úÖ **.dockerignore** = Skip node_modules/src
‚úÖ **NON-ROOT USER** = Security
‚úÖ **HEALTHCHECK** = Reliability
‚úÖ **ONE COMMAND** = docker run (no server config!)
‚úÖ **Image = FINAL ARTIFACT** ‚Üí Store in Harbor/ECR
