* `main` â†’ production
* `develop` â†’ integration
* `feature/*` â†’ development
* `hotfix/*` â†’ urgent prod fixes

## Git as Automation Trigger

| Git Action    | DevOps Result  |
| ------------- | -------------- |
| Commit        | CI starts      |
| Merge Request | Tests + review |
| Merge to main | Deployment     |
| Tag           | Release        |

Create release tag:

```bash
git tag v1.0.0
git push origin v1.0.0
```

---

## Rollback Using Git (Production Save)

```bash
git revert <bad-commit-id>
git push origin main
```

ðŸ‘‰ Pipeline redeploys stable version.

---

## DevOps Best Practices with Git

* Small, frequent commits
* Meaningful commit messages
* Mandatory Merge Requests
* Protected `main` branch
* `.gitignore` properly configured
* Never commit secrets
