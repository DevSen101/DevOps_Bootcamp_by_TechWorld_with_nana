### Undoing & Changing Commits (reset, revert, amend)

These commands are used to **fix mistakes safely** depending on whether commits are **local or already pushed**.

---

**Use when:** Commit is **local** and NOT pushed.

## 2. `git commit --amend` (Edit Last Commit)

**Use when:** Fix message or add missed files to **latest commit**.

Add forgotten file:

```bash

git commit --amend
```

Change commit message only:

```bash

```

ğŸ‘‰ Replaces last commit with a new one.

## 3. `git revert` (Safe Undo)

**Use when:** Commit is **already pushed**.

```bash

```

ğŸ‘‰ Creates a **new commit** that reverses changes.

**Why safe?**

* History is preserved
* Team-friendly

**One-liner:** Revert undoes changes without rewriting history.
